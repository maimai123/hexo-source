Vuex学习小计（麦麦）

初学vuex，总得弄懂他的思想。

vuex是什么？ 下面是官网给出的说法。
Vuex 是一个专为 Vue.js 应用程序开发的状态管理模式。它采用集中式存储管理应用的所有组件的状态，
并以相应的规则保证状态以一种可预测的方式发生变化。Vuex 也集成到 Vue 的官方调试工具 devtools extension，
提供了诸如零配置的 time-travel 调试、状态快照导入导出等高级调试功能。
总而言之，言而总之。就是用来管理你的state的！！！
如有转载，请注明出处。

我们从零开始。

可以参考官网 vuex 以及vuex文档 vuex-文档 也可以直接clone我的代码 麦麦的github

> 打开一个空的文件夹 dua~ 打开了吧？
> 安装 vue-cli。(确保你有node和npm)
> npm i -g vue-cli （只需安装一次，日后可省略这步）
然后创建一个webpack项目并且下载依赖
vue init webpack Test 进入 你创建的文件夹下 安装依赖
npm install 这些都准备好后，我们需要为我们的路由、vuex和XHR请求下载几个库，我们可以从vue的官网中找到他们。
npm i vue-resource vue-router vuex –save 如果你需要一个css编译器 你可以安装less-loader, vuejs-templates模板默认不安装less less-loader
npm install less less-loader –save-dev 接着使用 npm run dev 在热加载中运行我们的应用！！！ 打开浏览器应该可以看到我们的vue应用啦~
  • 我之前启动包了一个错 原因是 vue 的版本和 vue-template-compiler 的版本不一致 ，你可以在 package.json中 修改成相同的版本
  • 然后我们就可以开始修改文件啦~ 鸡冻不？
  • App.vue是一个类似于layout的文件。路由配置在router文件夹下
  • components文件夹下用来放你的组件
  • assets文件夹下存放你的静态资源
  • 我们新建一个vuex文件夹，这就开始学习状态管理啦~
  • 先新建 store.js 用来存放我们的状态
  • 再新建 getters 文件夹 用来处理state数据
  • 再新建 mutations 文件夹 用来注册我们各种数据变化的方法
  • 再新建 mutations-types.js 文件 用来记录我们所有的事件名(要写上备注哦 代码可不只是你一个人看的)
  • 再新建 actions 文件夹 可以编写异步的逻辑或者是一些逻辑，再去commit 我们的事件
  • 文件建好就开始开始写代码了
